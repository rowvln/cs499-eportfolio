# Enhancement Two
# Algorithms and Data Structures

The artifact selected for this enhancement is the Travlr Getaways full-stack travel application that I originally developed in CS 465. The application uses MongoDB, Express, Node.js, and Angular to provide both a customer-facing travel website and an administrative portal for managing trip information. I selected this artifact because it provides opportunities to demonstrate software engineering, database management, and algorithmic concepts within a single application. Since the project includes a frontend, backend API, and database, it allowed me to make meaningful improvements related to data processing, scalability, and application performance. I also felt that continuing to enhance a single project throughout CS 499 would provide a stronger demonstration of my growth than making smaller changes across multiple unrelated projects.

For this enhancement, I focused on improving how trip data is retrieved, processed, and displayed throughout the application. One of the biggest improvements was implementing backend search functionality using MongoDB text indexes. In the original version of the application, all trip records were retrieved from the database and returned to the frontend regardless of what the user was searching for. While this approach worked for a small dataset, it would become less efficient as the number of records increased. By implementing indexed search functionality, the database can now locate matching trips more efficiently and return only the results that meet the user's search criteria. I also added dynamic sorting functionality that allows trips to be sorted by multiple fields and sort directions directly within MongoDB queries. Moving this processing to the backend reduces unnecessary frontend work and creates a more scalable solution.

Another major enhancement was the implementation of pagination using MongoDB's skip and limit operations. Instead of returning every trip in the database during each request, the API now returns only the records needed for the current page. This reduces the amount of data transferred between the backend and frontend while improving response times and scalability. To further improve performance, I added database indexes that support search and sorting operations. These indexes allow MongoDB to retrieve information more efficiently and demonstrate the use of data structures to optimize application performance. Together, these enhancements improve efficiency, reduce unnecessary processing, and create a better overall user experience.

This enhancement directly supports the course outcome of designing and evaluating computing solutions using algorithmic principles and computer science practices. Through the implementation of search functionality, sorting, pagination, and database indexing, I evaluated different approaches for processing data and selected solutions that improve efficiency and scalability. The enhancement also supports the outcome of using well-founded techniques and tools to implement computing solutions that deliver value. By leveraging MongoDB indexing, query optimization, and reusable frontend services, I was able to improve the performance of the application while maintaining usability and functionality. In addition, the validation and secure API structure implemented during Enhancement One were preserved throughout this enhancement, ensuring that new functionality didn’t introduce unnecessary security risks.

One of the biggest lessons I learned from this enhancement was that algorithm optimization is not always about implementing complex algorithms from scratch. Improving database queries, reducing unnecessary processing, and using appropriate data structures can have a significant impact on application performance. One challenge I encountered was updating the application after changing the API response structure to support pagination. Once pagination metadata was added, portions of the application that previously expected a simple array of trip objects had to be updated to work with the new response format. Troubleshooting these issues reinforced the importance of understanding how changes in one layer of an application can affect other parts of the system. Overall, this enhancement significantly improved the scalability and efficiency of the Travlr application while allowing me to demonstrate practical applications of algorithms, data structures, database optimization, and full-stack software development principles.
