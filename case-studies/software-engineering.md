# Enhancement One
## Software Design and Engineering

The artifact I selected for the software design and engineering category was the Travlr Getaways application, which is a full-stack travel management platform originally developed during my CS 465 Full Stack Development class. The application uses Angular for the frontend, Node.js and Express for the backend, and MongoDB for database management. The project includes a customer-facing travel website, an administrative Angular application for trip management, RESTful API communication, authentication functionality, and CRUD operations connected to MongoDB.

I selected this artifact because it best demonstrates my ability to work across multiple layers of a full-stack application instead of focusing on only frontend or backend development individually. The project allowed me to showcase frontend development, backend API design, authentication, database communication, and software architecture within a single application. I also chose this artifact because it provided strong opportunities to improve maintainability, modularity, validation, secure coding practices, and overall software engineering quality.

For the enhancement, I focused on improving the structure, maintainability, security, and reliability of the application. One of the main enhancements involved implementing stronger validation throughout the backend API routes and controller logic. Additional validation was added to ensure trip data such as names, pricing, dates, descriptions, and trip codes were properly validated before being processed or stored in MongoDB. These changes improved defensive programming practices and reduced the risk of invalid or malformed data entering the system.

Another major enhancement involved improving centralized error handling throughout the backend application. Before enhancement, some controllers returned inconsistent error responses, which made debugging and long-term maintenance more difficult. To improve maintainability and scalability, I implemented reusable response handling logic to standardize API responses and reduce repeated code within the controllers. This helped improve organization, readability, and consistency throughout the backend structure.

I also strengthened authentication handling and route protection throughout the application. While Passport authentication was already implemented, I improved middleware handling and reinforced protection for administrative CRUD routes involving trip management. These enhancements helped strengthen secure coding practices and better protect backend resources from unauthorized access.

On the frontend, I improved Angular form validation and strengthened communication between frontend services and backend endpoints. Validation was added and improved within the add-trip and edit-trip forms to better handle invalid user input before requests reached the backend API. I also added visual required-field indicators using red asterisks and a short required-fields note at the top of each form so users could clearly understand which fields needed to be completed before submitting. These improvements increased reliability while also improving the overall user experience.

In addition to functionality improvements, I also focused on improving maintainability and readability throughout the application. Portions of the controller logic were refactored into more reusable and modular structures to reduce duplication and improve scalability. I also improved naming consistency, added documentation and comments where appropriate, and cleaned up portions of the codebase to improve readability for future developers.

This enhancement allowed me to demonstrate several course outcomes related to software design and engineering. Through this process, I demonstrated my ability to design and improve computing solutions using full-stack development principles while applying defensive programming and secure coding practices. I also strengthened my understanding of modular software architecture, validation workflows, API communication, middleware-based authentication, and scalable application organization. I met the course outcomes I planned to address by improving the application’s software design, strengthening validation and route protection, and applying secure coding practices across both the frontend and backend architecture.

One of the biggest things I learned during this enhancement process was how important maintainability and consistency become as applications grow larger. Even though the original application functioned correctly, working through the enhancement showed me how inconsistent validation, repeated logic, and weak error handling can create long-term scalability and maintenance issues. I also gained a stronger understanding of how backend validation, centralized error handling, and authentication middleware contribute to professional software engineering practices and application security.

One challenge I faced during the enhancement process involved maintaining compatibility between the Angular frontend and Express backend while refactoring portions of the application. Changes to validation logic, API responses, and frontend form behavior required careful testing to ensure the add-trip and edit-trip forms still worked correctly with the backend routes. Another challenge involved refactoring repeated logic without unintentionally affecting existing application functionality. Working through these challenges improved both my debugging skills and my understanding of full-stack application architecture.

Overall, the Travlr Getaways enhancement demonstrates my ability to improve an existing full-stack application using professional software engineering practices related to modularity, maintainability, validation, authentication, secure coding, and scalable application design.
